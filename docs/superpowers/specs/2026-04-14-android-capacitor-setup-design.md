# Android App Setup — Capacitor 7

## Overview

Set up the Reparilo web app as an Android application using Capacitor 7. The app is a single-location mobile phone repair shop management system, trilingual (AR/FR/EN), with RTL support for Arabic.

## Decisions

| Decision | Choice |
|---|---|
| Capacitor version | 7 (latest) |
| Package ID | `com.reparilo.app` |
| App name | Reparilo |
| Min SDK | 31 (Android 12) |
| Target SDK | 35 (Android 15) |
| Compile SDK | 35 |
| Native plugins | None initially (pure WebView) |
| API connection | Remote-only (no local server support) |
| Config format | `capacitor.config.ts` (typed) |

## 1. Capacitor Configuration

**File:** `capacitor.config.ts`

```typescript
import type { CapacitorConfig } from "@capacitor/cli";

const config: CapacitorConfig = {
  appId: "com.reparilo.app",
  appName: "Reparilo",
  webDir: "dist",
  server: {
    ...(process.env.NODE_ENV === "development" && {
      url: "http://localhost:5173",
      cleartext: true,
    }),
  },
  plugins: {
    SplashScreen: {
      launchAutoHide: false,
    },
  },
};

export default config;
```

- Development-only live reload URL (never committed to production).
- No server URL in production — app serves bundled assets, API calls go to remote server via existing Axios client.

## 2. Android Native Project Configuration

| Setting | Value |

| `hardwareAccelerated` | true |

- `android:hardwareAccelerated="true"` on `<application>`
- `android:largeHeap="true"` on `<application>`
- Internet permission (auto-added by Capacitor)

### ProGuard rules

Custom `proguard-rules.pro` to preserve Capacitor and React reflection classes.

## 3. Build & Development Workflow

| Command | Purpose |
|---|---|
| `pnpm build` | Vite build → `dist/` |
| `pnpm android` | `cap sync && cap open android` (existing) |
| `pnpm cap:sync` | Sync web assets + native plugins |
| `pnpm cap:copy` | Copy web assets only |

### Dev workflow

1. Run `pnpm dev` for live web development
2. Run `pnpm android` to open Android Studio with synced assets
3. For live reload on device: `capacitor.config.ts` dev server URL is active when `NODE_ENV=development`

### CI/CD build

```
pnpm build → npx cap sync → cd android && ./gradlew assembleRelease
```

## 4. API Connection Strategy

- Remote-only: Axios client in `src/lib/` points to configurable base URL
- Add `VITE_API_BASE_URL` env variable — defaults to empty (uses relative `/api` paths, proxied by Vite in web mode)
- Android build overrides with `VITE_API_BASE_URL=https://api.reparilo.com` (or equivalent)
- In Android WebView, all API requests go directly to remote server (no Vite proxy)

### Axios client adjustment

```typescript
const baseURL = import.meta.env.VITE_API_BASE_URL || "";
const api = axios.create({ baseURL: baseURL + "/api" });
```

## 5. Trilingual & RTL Handling

- Arabic locale sets `dir="rtl"` on `<html>` element (existing i18n logic)
- Android back button integration with React Router via Capacitor `App` plugin
- `StatusBar` plugin to adjust for RTL layout direction on locale change
- Ensure Android system UI respects RTL when Arabic is active

### App plugin for back button

```typescript
import { App } from "@capacitor/app";

App.addListener("backButton", ({ canGoBack }) => {
  if (canGoBack) {
    window.history.back();
  } else {
    App.exitApp();
  }
});
```

## 6. Security

- **Cleartext disabled** in production config (HTTPS only)
- No sensitive data stored in Capacitor `Preferences` — only non-UI preferences (theme, language)
- ProGuard enabled for release APKs (smaller size, basic obfuscation)
- WebView debugging disabled in release builds
- `android:usesCleartextTraffic="false"` in production manifest

## 7. Files to Create/Modify

| File | Action |
|---|---|
| `capacitor.config.ts` | Create |
| `android/` | Generated by `npx cap add android` |
| `package.json` | Add `@capacitor/core`, `@capacitor/cli`, `@capacitor/android` deps + scripts |
| `src/main.tsx` | Add back button handler, platform detection |
| `src/lib/` (Axios client) | Add `VITE_API_BASE_URL` support |
| `android/app/build.gradle` | Configure SDK versions, ProGuard |
| `android/app/src/main/AndroidManifest.xml` | Hardware accel, cleartext, permissions |
| `android/app/proguard-rules.pro` | Add preservation rules |
| `.env.example` | Add `VITE_API_BASE_URL` |
| `.gitignore` | `android/` already listed |

## 8. Implementation Order

1. Install Capacitor 7 dependencies
2. Create `capacitor.config.ts`
3. Run `npx cap add android` to scaffold Android project
4. Configure `android/app/build.gradle` (SDK versions, ProGuard)
5. Configure `AndroidManifest.xml` (hardware accel, cleartext)
6. Create ProGuard rules
7. Update Axios client with `VITE_API_BASE_URL`
8. Add Android back button handler in `src/main.tsx`
9. Add convenience scripts to `package.json`
10. Update `.env.example`
11. Build, sync, and verify in Android Studio
